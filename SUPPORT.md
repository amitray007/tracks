# Support

Tracks is pre-release software maintained on a best-effort basis.

## Questions and troubleshooting

Before opening an issue:

1. Read the [development and CLI instructions](README.md#development).
2. Run `pnpm tracks doctor` and review the output locally.
3. Search existing issues for the same behavior.

Open a bug report when you have a minimal synthetic reproduction. Use [GitHub Discussions](https://github.com/amitray007/tracks/discussions) for questions and early ideas, then open a feature request once a proposal is focused.

Do not post real sessions, credentials, private source code, unredacted paths, or access tokens. Reports involving a vulnerability or accidental exposure must use the private process in [SECURITY.md](SECURITY.md).

## Scope

Current support focuses on the Claude Code adapter, the local viewer and CLI, and the self-hosted Tracks server described in the repository. Other provider adapters remain future work unless the documentation says otherwise.
